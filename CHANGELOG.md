# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [0.11.0] - 2026-03-12 — Playwright E2E tests

### Added
- Playwright E2E test suite (`apps/web/tests/e2e/`) with browser-level tests:
  - `global-setup.ts` — authenticates as demo user, saves `storageState` for subsequent tests
  - `auth.spec.ts` — register, demo login, logout
  - `sets.spec.ts` — set grid loads, completion bars visible, click-through to set detail
  - `collection.spec.ts` — +1 increment updates qty, bulk clear removes items
- `e2e` job in GitHub Actions CI — runs after backend + frontend jobs pass; full-stack (Postgres + API + built Next.js)
- `make e2e` Makefile target
- `test:e2e` npm script in `apps/web/package.json`

### Fixed
- `api.ts` `request()` helper now handles 204 No Content responses

## [0.10.0] - 2026-03-12 — UI/UX

### Added
- Card image thumbnails in printings tables (set detail page); lazy-loaded with placeholder fallback
- Landing page hero section with live stats bar (fetches `/sets` to show real set and printing counts)

- "Try demo account" button on login page; "Try Demo" CTA on landing page
- `lib/theme.ts` — `useTheme()` hook (useSyncExternalStore), `toggleTheme()`, `setTheme()`

## [0.4.0] - 2026-03-03 — Collection

### Added
- `GET /collection/summary` — owned printings with full card and set detail; `?set_id=` scopes to one set
- `POST /collection/items` — upsert a single `{printing_id, qty}` pair; `qty=0` deletes the row
- `POST /collection/bulk` — atomic batch with actions `set_qty`, `increment`, `decrement`, and `clear`; all printing IDs are validated before any row is touched
- `app/login/page.tsx` and `app/register/page.tsx` — auth forms that store the access token and redirect to `/sets`
- `app/sets/page.tsx` — set grid with per-set completion bars (owned / total printings)
- `app/sets/[id]/page.tsx` — printings table with debounced name search, `+1` increment button, multi-select checkboxes, and bulk-action toolbar
- `lib/auth.ts` — `getToken` / `setToken` / `clearToken` over `localStorage` key `fab_access_token`
- `components/navbar.tsx` — navigation with auth-aware logout; `components/providers.tsx` — TanStack Query provider

### Changed
- Completion bars on the set grid update automatically after any ownership mutation via TanStack Query cache invalidation

## [0.3.0] - 2026-03-03 — Browse

### Added
- `GET /sets/{id}/printings` — printings in a set, paginated and filterable by `q`, `rarity`, `foiling`, `edition`, `hero_class`, `talent`, `card_type`
- `GET /search/printings` — cross-set printing search with all the same filters
- `SetSummary` schema: `printing_count` (always present) + `owned_count` (present only when authenticated)
- `get_optional_user` FastAPI dependency — `OAuth2PasswordBearer(auto_error=False)` returns `User | None` for optional-auth endpoints
- 23 new backend tests (13 sets + 10 search)

---

## [0.2.0] - 2026-03-03 — Card catalog

### Added
- `scripts/import_cards.py` (`make import-cards`) — downloads the pinned `the-fab-cube/flesh-and-blood-cards` release (~34 MB) and upserts sets, cards, and printings idempotently via `ON CONFLICT DO UPDATE`; imports ~92 sets, ~4,200 cards, ~14,000 printings
- `GET /cards` — paginated card list filterable by `name`, `hero_class`, `talent`, `pitch`, `set_code`
- `GET /cards/{id}` — card detail with all printings and their set information eager-loaded
- `GET /sets` — all sets with `printing_count`; adds `owned_count` when the caller is authenticated

---

## [0.1.0] - 2026-03-03 — Auth

### Added
- `POST /auth/register` — create an account; returns an access token + refresh token pair
- `POST /auth/token` — OAuth2 password-flow login
- `POST /auth/refresh` — rotate the refresh token (old token is immediately revoked)
- `POST /auth/logout` — revoke the refresh token server-side
- `GET /auth/me` — return the current user's profile
- `app/core/security.py` — `hash_password`, `verify_password` (bcrypt), `create_access_token`, `decode_access_token`
- `app/services/auth.py` — `create_refresh_token`, `use_refresh_token`, `revoke_refresh_token`
- `app/core/deps.py` — `get_current_user` FastAPI dependency
- 12 auth tests
