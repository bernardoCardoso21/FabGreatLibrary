## [0.5.0] - 2026-03-03 ‚Äî Missing printings & wishlists

### Added
- `GET /missing` ‚Äî paginated printings the authenticated user does not yet own; filterable by `set_id`, `card_id`, `edition`, `foiling`, `rarity`, `artists`
- `POST /wishlists` ‚Äî save a named filter set; returns HTTP 402 if the free-tier limit (1 wishlist per user) is already reached
- `GET /wishlists` ‚Äî list the current user's wishlists
- `DELETE /wishlists/{id}` ‚Äî delete a wishlist, re-opening the free-tier slot
- `app/missing/page.tsx` ‚Äî missing-printings page with set/foiling/rarity filters, paginated table, and wishlist save/load/delete panel
- "Missing" link in the navbar
- 17 new backend tests (11 wishlist + 6 missing); total 92 passing

## [0.4.0] - 2026-03-03 ‚Äî Collection

### Added
- `GET /collection/summary` ‚Äî owned printings with full card and set detail; `?set_id=` scopes to one set
- `POST /collection/items` ‚Äî upsert a single `{printing_id, qty}` pair; `qty=0` deletes the row
- `POST /collection/bulk` ‚Äî atomic batch with actions `set_qty`, `increment`, `mark_playset` (qty=3), and `clear`; all printing IDs are validated before any row is touched
- `app/login/page.tsx` and `app/register/page.tsx` ‚Äî auth forms that store the access token and redirect to `/sets`
- `app/sets/page.tsx` ‚Äî set grid with per-set completion bars (owned / total printings)
- `app/sets/[id]/page.tsx` ‚Äî printings table with debounced name search, `+1` increment button, multi-select checkboxes, and bulk-action toolbar
- `lib/auth.ts` ‚Äî `getToken` / `setToken` / `clearToken` over `localStorage` key `fab_access_token`
- `components/navbar.tsx` ‚Äî navigation with auth-aware logout; `components/providers.tsx` ‚Äî TanStack Query provider

### Changed
- Completion bars on the set grid update automatically after any ownership mutation via TanStack Query cache invalidation

## [0.3.0] - 2026-03-03 ‚Äî Browse

### Added
- `GET /sets/{id}/printings` ‚Äî printings in a set, paginated and filterable by `q`, `rarity`, `foiling`, `edition`, `hero_class`, `talent`, `card_type`
- `GET /search/printings` ‚Äî cross-set printing search with all the same filters
- `SetSummary` schema: `printing_count` (always present) + `owned_count` (present only when authenticated)
- `get_optional_user` FastAPI dependency ‚Äî `OAuth2PasswordBearer(auto_error=False)` returns `User | None` for optional-auth endpoints
- 23 new backend tests (13 sets + 10 search)

---

## [0.2.0] - 2026-03-03 ‚Äî Card catalog

### Added
- `scripts/import_cards.py` (`make import-cards`) ‚Äî downloads the pinned `the-fab-cube/flesh-and-blood-cards` release (~34 MB) and upserts sets, cards, and printings idempotently via `ON CONFLICT DO UPDATE`; imports ~92 sets, ~4,200 cards, ~14,000 printings
- `GET /cards` ‚Äî paginated card list filterable by `name`, `hero_class`, `talent`, `pitch`, `set_code`
- `GET /cards/{id}` ‚Äî card detail with all printings and their set information eager-loaded
- `GET /sets` ‚Äî all sets with `printing_count`; adds `owned_count` when the caller is authenticated

---

## [0.1.0] - 2026-03-03 ‚Äî Auth

### Added
- `POST /auth/register` ‚Äî create an account; returns an access token + refresh token pair
- `POST /auth/token` ‚Äî OAuth2 password-flow login
- `POST /auth/refresh` ‚Äî rotate the refresh token (old token is immediately revoked)
- `POST /auth/logout` ‚Äî revoke the refresh token server-side
- `GET /auth/me` ‚Äî return the current user's profile
- `app/core/security.py` ‚Äî `hash_password`, `verify_password` (bcrypt), `create_access_token`, `decode_access_token`
- `app/services/auth.py` ‚Äî `create_refresh_token`, `use_refresh_token`, `revoke_refresh_token`
- `app/core/deps.py` ‚Äî `get_current_user` FastAPI dependency
- 12 auth tests
